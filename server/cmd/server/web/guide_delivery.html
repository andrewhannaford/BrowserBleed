Delivery

Get the payload to the target using smart links, calendar invites, or direct email delivery.

Smart links

A smart link is a URL of the form /p/{preset} that detects the visitor's OS from their User-Agent and automatically serves the correct payload. The file downloads with a disguised name matching the preset (e.g. chrome_crashpad_handler.exe for the chrome preset on Windows). 

https://reports.yourdomain.com/p/chrome
https://reports.yourdomain.com/p/slack
https://reports.yourdomain.com/p/zoom

Smart links require at least one uploaded payload for the target OS. If no matching payload exists, the link returns 404. Build and upload payloads via the Payloads page first.

Copy a smart link from the Smart Link column in the payload table, or from the Calendar Invite panel after selecting a preset.

Calendar invite

A calendar invite is the most effective delivery method. The smart link is embedded in the meeting body — appearing as a Zoom join link, Teams companion download, or Google Meet materials link, depending on the disguise you choose. The target clicks what looks like a normal pre-meeting resource and the payload downloads silently.

Workflow

  1. Go to Payloads → Calendar Invite and select a meeting type preset.
  2. Fill in the invite details: sender name, sender email, recipient, subject, date/time, and duration. The subject pre-fills with a plausible meeting title for the preset.
  3. Choose a disguise — controls the branding of the invite: Zoom, Microsoft Teams, Google Meet, or a plain Generic style. The smart link is embedded in all disguises.
  4. Deliver: click Download ICS to get the invite file for manual sending, or click Send Invite to deliver via your connected email account.

ICS download vs. Send Invite: The Download ICS button generates the invite file in your browser using JavaScript — no server call. Send Invite sends from the server using your connected email account, which keeps the sender's address consistent with the From Name field.

Email setup

To use Send Invite, connect a Gmail or Outlook account, or configure a custom SMTP server. Go to Payloads → Email Settings.

Gmail (recommended)

  1. Go to Google Cloud Console → APIs & Services → Credentials.
  2. Create an OAuth 2.0 Client ID (Web application type).
  3. Enable the Gmail API for your project.
  4. Add this redirect URI: https://your-server.com/auth/google/callback
  5. In Email Settings → OAuth app credentials, enter your Client ID and Secret, then click Save Credentials.
  6. Click Sign in next to Gmail. Google will redirect you back after authorization.

Outlook / Office 365

  1. Go to Azure Portal → Entra ID → App registrations → New registration.
  2. Under API permissions, add delegated permissions: Mail.Send, User.Read, offline_access.
  3. Under Authentication, add this redirect URI (Web platform): https://your-server.com/auth/microsoft/callback
  4. Create a client secret under Certificates & secrets.
  5. In Email Settings → OAuth app credentials, enter your Client ID and Secret, then click Save Credentials.
  6. Click Sign in next to Outlook.

Custom SMTP (fallback)

SMTP is used only if no OAuth provider is connected. Expand Custom SMTP in Email Settings and fill in your SMTP credentials.

For Gmail SMTP: enable 2-Factor Authentication on your Google account, then generate an App Password (Google Account → Security → App Passwords). Use smtp.gmail.com on port 587.

OAuth is preferred over SMTP. Gmail and Office 365 block plain SMTP auth on many accounts by default. OAuth uses the modern REST API (Gmail API / Graph API) and is much more reliable.

Tips

Multiple recipients: The To field accepts a single email address. For multiple targets, send separate invites so each gets an individualized invite and you can track which target triggered which download.

From name: The From Name field controls the display name in the invite. Pick a plausible sender — someone the target would expect a meeting invite from (a manager, an IT contact, a vendor).

Delivery timing: Calendar invites sent a few days ahead look more legitimate than same-day invites. Use the date/time picker to schedule the fake meeting for a realistic future time.

Fallback if no OAuth: If you have not connected Gmail or Outlook and have no SMTP configured, the Send Invite button will return an error. Download the ICS file and attach it manually to an email instead.
