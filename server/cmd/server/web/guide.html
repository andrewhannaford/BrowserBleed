BrowserBleed Reports Payloads Guide

sessiontest.py

Validate credentials and cookie sessions captured by BrowserBleed against live services.

For authorized red team use only. Only run against credentials you are authorized to test.

Overview

sessiontest.py reads a bb_results.csv produced by BrowserBleed and attempts to authenticate to each identified service. It reports the identity and access level for each credential that is still valid.

API tokens are identified by their prefix (e.g. ghp_ for GitHub, sk-ant- for Anthropic). Cookie sessions are grouped by domain and tested against known per-domain auth endpoints. AWS key pairs are signed with SigV4 and verified via STS. JWTs are decoded locally and checked for expiry.

Prerequisites

Python 3.10+ is required. No third-party packages are needed for basic use.

The --browser flag requires Playwright:

pip install playwright
playwright install chromium

Usage

python3 sessiontest.py [csv_file] [options]

Use the Session Test command builder on the Payloads page to generate the exact command for your OS.

Cookie sessions

Cookies are grouped by domain. For known domains, a specific auth endpoint is tested. For unknown domains, the tool compares the HTTP status with vs. without cookies to detect redirect-to-login behaviour.

CSV format

bb_results.csv is generated by BrowserBleed automatically. Relevant columns used by sessiontest:

ColumnDescription
labelCredential type (e.g. GitHub Token, Cookie). Rows with "cookie" in the label are treated as cookie sessions.
serviceService name or domain (e.g. github.com). Used as the domain for cookie grouping.
addressFallback domain for cookies if service is empty.
valueThe credential value. For cookies, formatted as name=value.

Status meanings

StatusMeaning
✓ validCredential accepted, identity and access resolved.
⏱ expiredJWT is past its exp claim, or token was revoked.
✗ invalidRejected with 401 or 403.
! errorNetwork failure or unexpected response.
· infoDecoded locally (JWT) or cookie session could not be confirmed without browser.

Examples

Quick test — auto-find CSV

python3 sessiontest.py

Explicit file path

python3 sessiontest.py /path/to/results.csv

Open browser for cookie sessions (requires Playwright)

python3 sessiontest.py --browser

Launches a Chromium window with the captured cookies pre-loaded. You can manually verify the session and inspect what the victim account can access.

JSON output — filter with jq

python3 sessiontest.py --json | jq '.[] | select(.status == "valid")'
python3 sessiontest.py --json | jq '[.[] | select(.service == "GitHub")]'

Slower requests to avoid rate-limits

python3 sessiontest.py --timeout 20 --delay 1.0

Skip TLS verification (proxy / internal env)

python3 sessiontest.py --no-verify-ssl

Windows

python sessiontest.py
python sessiontest.py results.csv --browser --timeout 15

Tips

Deduplification: Non-cookie credentials with the same value are tested only once, even if they appear multiple times in the CSV.

AWS key pairing: The tool looks for a single AKIA… / ASIA… access key and a single matching secret key (label AWS Secret Key or AWS Secret Access Key, 40 chars). If found as a pair, it signs an STS request. If the secret is missing, the row is skipped with an info status.

Unknown cookie domains: Any domain not in the known list falls back to a 200-vs-redirect heuristic. This is accurate for most SaaS apps but may give false negatives on apps that return 200 even for unauthenticated visitors. Use --browser to confirm.
